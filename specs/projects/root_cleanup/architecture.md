# Architecture: Root Cleanup

Most items are mechanical file moves. This doc focuses on the three places where mechanics get subtle, plus the cross-cutting conventions used across phases.

## Cross-cutting conventions

### Validation pattern per phase

Every phase ends with:
1. **Automated:** `uv run ./checks.sh --agent-mode` (lint, format, types, python tests, web lint/check/test, schema check).
2. **Manual spot-check:** phase-specific (listed inline below).
3. **Commit + PR.** Small PRs, one phase per PR.

### Git move mechanics

- Use `git mv` (not `mv`) so git tracks the rename. Reduces diff noise and preserves blame.
- When deleting a file that was tracked, `git rm` it in the same commit as any replacement edits.

### Reference-update discipline

For each move, grep for the old path across the entire repo **including hidden dirs** (`.github/`, `.config/`, `.agents/`, `.cursor/`, `docs/`, `guides/`). Don't miss:
- CI workflows (`.github/workflows/*.yml`)
- Docs (`CONTRIBUTING.md`, `README.md`, `AGENTS.md`, `.agents/*.md`)
- Shell scripts (`checks.sh`, `Makefile`, and anything in `.config/utils/` after it moves)
- Package manifests (`pyproject.toml`, `package.json`)

## Subtle area 1 — `.cursor/skills/` becomes generated (item 8)

### Problem

Today `.cursor/skills/*` is tracked. Tomorrow it's generated by a setup script from `.agents/skills/`. Git won't automatically stop tracking files just because we add them to `.gitignore` — they must be explicitly removed from the index.

### Approach

Phase order inside item 8:
1. Create `.agents/skills/` and copy tracked content from `.cursor/skills/`.
2. Fix hardcoded path refs inside the moved files (`kiln-check-deprecation/SKILL.md` × 2).
3. `git rm -r --cached .cursor/skills` (remove from index, keep on disk for current dev).
4. Add `.cursor/skills/` to `.gitignore`.
5. Update setup script (`.agents/cursor/setup.sh`) to copy `.agents/skills/*` → `.cursor/skills/*`.
6. Run the setup script in this worktree to confirm it regenerates `.cursor/skills/` identically.
7. Verify with `diff -r .agents/skills .cursor/skills` → empty output.

### Risks

- Cursor might cache the old location. Restart Cursor after the switch.
- The same approach is needed for `.claude/skills/` except `.claude/` is already gitignored, so only the setup-script edit matters for Claude.

### Manual validation

- Open Cursor; verify skills list populates (try invoking `kiln-add-model` or `kiln-check-deprecation`).
- Open Claude Code in this worktree; verify it can list/run the same skills.

## Subtle area 2 — `hooks_mcp.yaml` prompt path resolution (item 9)

### Problem

`hooks_mcp/config.py:328` resolves prompt file paths relative to the config file's parent directory. After the move, that directory changes from repo root to `.config/`. Every prompt path in the YAML breaks unless we either (a) rewrite them with `../` prefix, or (b) teach the server to use `--working-directory` for prompt resolution.

### Approach

Take (a) — mechanical rewrite. Every prompt path in `.config/hooks_mcp.yaml` gets a leading `../`:

```yaml
# Before
prompts:
  - name: agents
    file: AGENTS.md

# After
prompts:
  - name: agents
    file: ../AGENTS.md
```

Apply the same transformation to every prompt path (all start from repo root today).

### Risks

- If the user has unsaved changes to `hooks_mcp` that change this behavior, the transformation becomes wrong. Mitigation: have user confirm the current behavior before starting.
- `.cursor/mcp.json` change is small but critical — Cursor needs a restart to reload MCP config.

### Manual validation

- Restart Cursor.
- In a Cursor session, confirm HooksMCP tools appear (`check_all`, `test_file_python`, etc.).
- Invoke one MCP prompt (e.g., the `AGENTS.md` prompt) and confirm it returns the file content.
- Run `check_all` via MCP and confirm it executes `uv run ./checks.sh --agent-mode` at repo root correctly.

## Subtle area 3 — `tests/assets/` fixture migration (item 11)

### Problem

Root `conftest.py` exposes fixtures whose asset directory resolves via `Path(__file__).parent / "tests" / "assets"`. Moving assets to `libs/core/tests/assets/` changes that base. Additionally, pytest only auto-loads `conftest.py` files that sit in the collected path ancestry — so a new `libs/core/conftest.py` only applies to tests under `libs/core/`. The one server consumer (`libs/server/kiln_server/test_document_api.py`) needs a bridge.

### Approach

1. Move asset files: `git mv tests/assets/ libs/core/tests/assets/`.
2. Create `libs/core/conftest.py`:
   ```python
   from pathlib import Path
   # ... existing MockFileFactoryMimeType enum ...

   @pytest.fixture
   def test_data_dir() -> Path:
       return Path(__file__).parent / "tests" / "assets"

   # ... mock_file_factory, mock_attachment_factory fixtures ...
   ```
3. Create `libs/server/conftest.py` that re-exports (approach chosen for clarity over inlining):
   ```python
   from libs.core.conftest import mock_file_factory, mock_attachment_factory  # noqa: F401
   ```
   **Note:** validate that import path resolves. If not (monorepo import layout may require `kiln_ai.conftest` or a pytest rootdir adjustment), fall back to inlining a minimal duplicate in `libs/server/conftest.py`. Decide based on what `uv run pytest libs/server/kiln_server/test_document_api.py -q` reports.
4. Remove the moved fixtures from root `conftest.py`. Keep any remaining repo-wide fixtures (autouse `reset_config`, `use_temp_settings_dir`, CLI options `--runpaid`, `--runslow`, `--ollama`).
5. `git rmdir` (conceptually — just remove the now-empty `tests/` folder).

### Risks

- `conftest.py` auto-discovery rules differ subtly from module imports. Sibling `conftest.py` can't import from another sibling `conftest.py` via package path if `libs/core/` isn't on `sys.path`. If the re-export fails, the fallback is inlining.
- Any test currently relying on the root-level `test_data_dir` from outside `libs/core` / `libs/server` will break. Unlikely given the grep result (only 9 consumers, all accounted for), but confirm with a fresh grep just before executing.

### Manual validation

- `uv run pytest libs/core -q -n auto` passes.
- `uv run pytest libs/server -q -n auto` passes.
- `uv run pytest -q -n auto .` (full suite) passes with identical test count to pre-move.
- `rg -n 'test_data_dir|mock_file_factory|mock_attachment_factory' --hidden` shows only: `libs/core/conftest.py` (definitions), `libs/server/conftest.py` (re-export), and the 9 consumer test files.
