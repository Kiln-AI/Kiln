# Phase 6: Consolidate skills at `.agents/skills/`

## Overview

Move the tracked skills from `.cursor/skills/` into a single canonical source at `.agents/skills/`. `.cursor/skills/` becomes a generated artifact (regenerated by a setup script in Phase 7), paralleling how `.claude/` is handled today. Gitignore `.cursor/skills/` so it stops being tracked, and update hardcoded script path references inside `kiln-check-deprecation/SKILL.md` so they point at the new canonical location.

Rationale: one source of truth for skills. Avoids drift between `.cursor/skills/` and `.claude/skills/`, and positions the repo for Phase 7 where both Cursor and Claude setup scripts will copy from `.agents/skills/`.

## Steps

1. `git mv .cursor/skills .agents/skills` to move the tracked tree. This preserves blame and removes `.cursor/skills/` from the index in one operation.
2. Edit `.agents/skills/kiln-check-deprecation/SKILL.md`: replace the three `.cursor/skills/kiln-check-deprecation/scripts/...` path references with `.agents/skills/kiln-check-deprecation/scripts/...` (lines 27, 52, 59).
3. Add `.cursor/skills/` to the root `.gitignore` next to the existing `.claude/` line.
4. Verify `git status` shows `.cursor/skills/` is no longer tracked (deleted in index) and `.agents/skills/` now holds the four files:
   - `kiln-add-model/SKILL.md`
   - `kiln-check-deprecation/SKILL.md`
   - `kiln-check-deprecation/scripts/check_provider.py`
   - `kiln-check-deprecation/scripts/extract_models.py`
5. Do not touch `.claude/` (already gitignored; regenerated by `setup_claude.sh`).
6. Run `uv run ./checks.sh --agent-mode`.

## Tests

- No new tests; this is a file move + gitignore edit + text fix. Validation is:
  - `git ls-files .agents/skills/` lists the four expected files.
  - `git ls-files .cursor/skills/` returns nothing.
  - `grep -r '.cursor/skills/kiln-check-deprecation/scripts' .agents/skills/` returns nothing.
  - `uv run ./checks.sh --agent-mode` passes.
