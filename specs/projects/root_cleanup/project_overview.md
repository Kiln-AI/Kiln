# Root Cleanup

Reduce top-level clutter in the Kiln monorepo. The root has too many files/folders, pushing the README down and making the repo feel disorganized. This project moves, consolidates, or removes root files where it's safe to do so.

## Goals

- Fewer entries at the repo root.
- No loss of functionality (CI, linting, coverage, agent tooling, Worktrunk, VS Code).
- Prefer small, reviewable phases over one mega-PR.
- Accept some SEO loss on legacy `guides/` URLs; the benefit of a clean root outweighs it.

## In scope

- Moving/deleting root files and folders listed in the decisions table below.
- Updating references (CI, scripts, docs) to match new paths.
- Updating setup scripts (`setup_claude.sh`, `setup_env.sh`) to match new layout.

## Out of scope

- Restructuring `libs/`, `app/`, or `specs/`.
- Renaming anything that affects public GitHub URLs we care about (we're explicitly accepting SEO loss on `guides/`).
- Touching generated files (`uv.lock`, `.venv/`).
- `.claude/` folder and `CLAUDE.md` layout — these are already gitignored and generated by `setup_claude.sh`. Don't restructure their on-disk form; only update the setup script to point at the new central skill location.
