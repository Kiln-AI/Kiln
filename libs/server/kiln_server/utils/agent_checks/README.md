# Agent Policy Checks

This module controls which API endpoints an AI agent can call, which require user approval, and which are blocked.

## Annotations

Every API endpoint is annotated with an `AgentPolicy` via FastAPI's `openapi_extra` parameter using one of three helpers:

- `ALLOW_AGENT` -- agent can call freely
- `DENY_AGENT` -- agent cannot call
- `agent_policy_require_approval("reason")` -- agent can initiate, but the user must approve

## `annotations/` folder

The `annotations/` folder contains one JSON file per API endpoint, generated by the dump CLI (`dump_annotations.py`). These files are checked into git and verified by CI.

The folder intentionally retains annotation files for older/removed endpoints. An agent may be talking to multiple versions of the Kiln client API, so stale annotations are needed to serve older clients.

Each annotation file carries:

- `agent_policy` — required. Missing/invalid policy fails the dump.
- `request_body` — `null` for bodyless methods, otherwise `{required, content_type, schema}`. The `schema` is a JSON Schema dereferenced against `components.schemas` (best-effort; missing schemas log a warning and yield `null`).
- `parameters` — always `{"path": {...}, "query": {...}}`, each keyed by parameter name. `header`/`cookie` parameters are ignored.

When a request body references a self-recursive or mutually-recursive Pydantic model, the `schema` is emitted with a local `$defs` map and back-edges rewritten to `{"$ref": "#/$defs/<ModelName>"}`. Consumers running `jsonschema.validate` should use a Draft 2020-12 validator (or any validator that resolves `#/$defs/...`) so local refs resolve within the same document.

To regenerate after adding or changing endpoints:

```
uv run python -m kiln_server.utils.agent_checks.dump_annotations http://localhost:8757/openapi.json libs/server/kiln_server/utils/agent_checks/annotations
```

## Policy Lookup

Agents should use `AgentPolicyLookup` to check endpoint permissions at runtime:

```python
from kiln_server.utils.agent_checks import AgentPolicyLookup, AgentPolicyError

lookup = AgentPolicyLookup("libs/server/kiln_server/utils/agent_checks/annotations")
policy = lookup.get_policy("GET", "/api/projects")

if policy.permission == "deny":
    # blocked
elif policy.requires_approval:
    # show policy.approval_description to user and wait for confirmation
else:
    # allowed
```

Unknown or unannotated endpoints raise `AgentPolicyError` (fail-safe: blocked by default).
